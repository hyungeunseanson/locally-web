## [Phase 1] 회원가입 및 관리자 대시보드 연동 QA
**Target Flow**: 신규 유저 가입 → Supabase Auth & DB 동기화 → 관리자 페이지 실시간 노출
**Date**: 2026-03-05

### 1. 테스트 항목 및 결과
- [x] **[모달 UI]** 성별 선택 드롭다운 UI 스타일 일관성 점검 ➡️ 수정 완료
- [x] **[모달 기능]** 이름 우측에 국적 선택창 배치 및 유효성 검사 적용 ➡️ 수정 완료
- [x] **[모달 기능]** 필수(이용약관, 개인정보) 동의 체크박스 로직 추가 및 마케팅(선택) 제거, 모달 형식의 약관 보기 UI 구현 ➡️ 수정 완료
- [x] **[DB 트랜잭션]** Supabase Trigger에서 신규 유저 생성 시 `nationality` 필드 누락 현상 점검 ➡️ 수정 완료 (`handle_new_user` 함수 업데이트)
- [x] **[관리자 연동]** 대시보드(`useAdminData.ts`) 내역에 신규 가입자가 실시간(Realtime) 노출되지 않는 버그 점검 ➡️ 수정 완료 (Realtime Channel `profiles` 추가)
- [x] **[유저 설정 연동]** 어카운트 세팅(`Account` 페이지) 뷰에서 수정된 국적 데이터 렌더링 및 달력 날짜 반영 이슈 점검 ➡️ 수정 완료

### 2. 이슈 및 해결
* **이슈**: 신규 가입 이벤트 발생 시 관리자 페이지에 유저 리스트가 즉각 새로고침되지 않음.
* **해결**: `useAdminData.ts`에 `profiles` 테이블의 `INSERT` 이벤트를 감지하는 Supabase Realtime 채널을 활성화하고 토스트 메시지와 함께 `setState` 배열 맨 앞단에 새 유저를 주입함.
* **이슈**: 생년월일 달력 날짜 설정 시, 기본 연도가 올바르게 표시되지 않거나 연도 이동이 번거로운 문제.
* **해결**: `Account` 페이지 커스텀 달력 모달의 렌더링 방식 개선 및 기본 설정 날짜를 2000년으로 지정.

### 3. 향후 논의 및 보완 스펙
* **휴대폰 본인 인증**: 현재 통신사/PG 연동 시 건당 과금(15~50원) 발생 및 글로벌 유저 접근 불가 이슈가 있음. 대체 방안으로 이메일 인증(Supabase OTP 기반) 또는 첫 결제/호스트 등록 과정으로 본인 인증 요구를 미루는 방안을 검토 권장.
