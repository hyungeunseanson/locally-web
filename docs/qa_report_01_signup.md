# QA E2E Report: 회원가입 및 관리자 대시보드 연동 플로우

## 1. 개요
* **테스트 대상**: 웹 로그인/회원가입 모달 -> Supabase Auth & Profiles 데이터베이스 -> 관리자 대시보드 연동
* **목적**: 신규 유저 가입 시의 프론트엔드 예외 처리, 데이터베이스 트랜잭션 안전성, 관리자 페이지 실시간 반영 여부를 검증.

## 2. 테스트 과정 및 보완 사항 (AI 선행 검증)

### 2.1 프론트엔드 (회원가입 모달 - `LoginModal.tsx`)
* **점검 사항**: 유효성 검사, 로딩 상태 시 중복 클릭 방지, UI 일관성.
* **이슈 발견**:
  1. 성별 선택(`<select>`)의 폰트 스타일이 일반 텍스트 입력창과 달라 일관성이 떨어짐.
  2. 요구사항: 회원가입 폼에 **국적 선택**이 필요함.
  3. 요구사항: 필수/선택 **약관 동의 체크박스** 누락.
* **보완 완료**:
  - 선택되지 않았을 때(placeholder 상태) 텍스트를 `text-transparent`로 처리해 기본 placeholder처럼 동작하고 값이 있을 때 `text-gray-900`으로 나타나도록 스타일 수정.
  - 이름(실명) 입력란 우측에 동일한 디자인 컨벤션으로 **국적 선택(`<select>`)** 드롭다운 UI 추가.
  - 폼 하단에 전체동의, 서비스 이용약관, 개인정보 수집 등 **필수/선택 약관 체크박스 컨테이너** 추가. 미 체크시 에러 토스트와 함께 UI가 붉게 변하도록 유효성 검사 강화.

### 2.2 백엔드 및 데이터베이스 (`supabase_profile_sync_v3_migration.sql`)
* **점검 사항**: Auth 생성 후 Profiles 테이블로 데이터 삽입 시의 트랜잭션 안전성 (`handle_new_user` 트리거).
* **이슈 발견**: 모달에서 추가된 `nationality` 데이터가 `raw_user_meta_data`로는 들어가지만 트리거가 해당 컬럼을 `profiles`에 삽입해주지 않음.
* **보완 완료**: 
  - `handle_new_user()` 함수를 수정하여 `NEW.raw_user_meta_data->>'nationality'` 값을 `profiles`의 `nationality` 컬럼에 올바르게 삽입하도록 수정 완료.

### 2.3 관리자 대시보드 & 계정/프로필 관리 (`useAdminData.ts`, 계정 페이지)
* **점검 사항**: 
  1. 새 유저 가입 시 관리자 유저 리스트에 실시간 반영되는가?
  2. 새로 생성된 국적 정보가 계정 관리(Account) 및 프로필 설정에서 잘 렌더링되고 수정 가능한가?
* **이슈 발견**:
  1. 관리자 대시보드에 예약 등은 실시간 반영되나, `profiles`에 대한 실시간 구독 채널이 없어 새로고침 전에는 신규 가입자가 노출되지 않음.
* **보완 완료**:
  - `useAdminData.ts`에 `realtime_profiles` 채널 구독 로직을 추가, 새 유저 가입 시(`INSERT` 이벤트) `users` 상태 배열 맨 앞에 실시간으로 항목이 추가되게 보완 완료.
  - `app/account/page.tsx` 등 관련 계정 관리, 프로필 뷰에서 `nationality`가 정상적으로 `<select>` 폼 및 통계에 연동되어 렌더링되며 수정 가능함을 확인함.

## 3. 휴대폰 번호 인증 기능 구현 방안 (의견)

휴대폰 본인 인증 로직(SMS OTP, 통신사 PASS 등)은 통신망을 이용하거나 KCB/나이스평가정보 등의 신용정보 시스템을 거치므로 **기본적으로 과금이 발생하는 서비스(건당 약 15~50원+)**이며, 완전 무료로 제공할 수 있는 API는 존재하지 않습니다. 다만 운영 및 개발 관점에서 다음과 같은 구현 방안들을 검토해볼 수 있습니다.

### 제안 1: 국내 통신사 통합 본인인증 (가장 안전 / 비용 발생)
* **방식**: 포트원(PortOne), 토스페이먼츠 등의 결제/인증 PG 라이브러리를 통해 통신사 PASS 인증을 연동.
* **장점**: 가장 확실한 1인 1계정 통제가 가능. 입력받은 실명/생년월일/휴대폰 번호가 국세청 및 신용평가원 정보와 100% 일치하는지 검증하므로 어뷰징(다중계정, 미성년자)을 원천 차단함.
* **단점**: PG사 가입비/연회비 등 초기 비용, 1건 당 40~50원의 높은 수수료. 또한 글로벌 유저(해외 통신사)는 인증 불가능.

### 제안 2: SMS 기반 단순 OTP 인증 (글로벌 범용성 / 저비용)
* **방식**: Supabase Phone Auth 또는 Twilio, 알리고(Aligo), 쿨에스엠에스(CoolSMS) 등의 API를 연결해 유저 핸드폰으로 단순 인증번호 6자리를 보내고 검증함.
* **장점**: SMS 1건 당 약 10~15원 수준으로 상대적인 저비용 구축. 글로벌 번호로도 발송 가능.
* **단점**: 이 번호가 과연 유저 본인의 명의가 맞는지 (대포폰 여부)까진 알 수 없음.

### ⭐️ 제안 3: 초기 스타트업을 위한 무료/저비용 대안 ⭐️ (추천)
무조건 첫 회원가입 문턱부터 휴대폰 인증 허들을 올리면 이탈률이 급증합니다. 따라서 초기엔 아래 방식을 추천합니다:
1. **소셜 로그인 (Oauth) 강력 활용**: 네이버, 카카오 등 이미 본인인증을 통과한 계정 시스템과 연결 (현재 시스템에서 권장).
2. **이메일 인증(Email Verification) 활용**: 휴대폰 번호는 단순 텍스트로만 받아두고(로컬 매칭 및 결제 참고용), 실제 보안 검증은 가입 시 **무료로 쏠 수 있는 Supabase 확인 메일 링크 클릭**에 의존합니다.
3. **결제/정산 시점 인증 검증 시프트**: "회원가입" 단계가 아닌 실제 신뢰도가 필요한 지점, 즉 카드로 결제를 시도할 때(PG사에서 본인 확인 진행), 혹은 호스트로 신청할 때에만 계좌주와 실명을 비교하는 방식으로 우회하여 가입 장벽과 비용을 모두 낮춥니다.
