# Email Notification Policy

## 🎯 현재 기준

- ✅ 정말 중요한 상태 변경은 즉시 이메일
- 🔔 나머지는 인앱 알림 우선
- ⏸️ cron이 필요한 지연/묶음 메일은 현재 보류

---

## 📌 현재 운영 정책

### 즉시 이메일

- 💳 예약 결제 완료
- ❌ 예약 취소 완료
- 🛠️ 호스트 신청 결과
  - 승인
  - 보완 요청
  - 거절
- 📥 무통장 입금 확인 완료
- 📞 프록시 예약 댓글/답변
- 💬 문의 새 메시지
- 👥 팀 메모 / 메모 답글

### 인앱만

- ⭐ 새 후기 등록
- ✍️ 후기 작성 요청
- 🧭 서비스 매칭 일반 상태 변경
  - 새 지원자 등록
  - 다른 호스트 선택됨
  - 서비스 취소 요청 접수

---

## 🚨 보류된 정책

아래 방향은 맞지만, 지금은 적용하지 않는다.

- 💬 문의 `미읽음 10분 후 1회 메일`

보류 이유:

- GitHub Free 비공개 저장소의 Actions minutes 한도와 맞지 않음
- 별도 스케줄러 없이 운영하면 비용 대비 효율이 낮음

현재 예외:

- 🗨️ 팀채팅은 `읽지 않은 배치당 첫 메일 1회` 방식으로만 발송
- ✅ TODO / 할 일 댓글은 메일 미발송
